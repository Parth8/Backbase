Error Mapping Config

Version 3.5  ·  Meridian Bank NV  ·  gc_error_mapping_config_v3.json

Maps a raw error code returned by the rail, connector, or core to a three-field canonical envelope. Every entry is deterministic - origin, code, and reason are derived from the config match, not from runtime state.

origin
Where the failure came from. RAIL, CONNECTOR, CORE, or UNKNOWN.
code
Canonical code. Stable across all banks on the platform.
reason
Plain English description for the ops team.
What this config does not contain: retry flags, SLA targets, ops escalation paths. Those are the bank's operational policy, not the connector's contract.

Exceptions generated by the connector itself, including timeouts, serialisation errors, and connection failures. The timeout is just another code - no special playbook. Platform-standard across all banks.
› Core Bank core codes  ·  Bank-owned 33 entries
Meridian Bank's Temenos T24 error codes. This section is owned and maintained by Meridian's integration team. Any bank using this connector maintains their own equivalent section.
