Payments Connector

Version 2.0.0  ·  Sparkasse Mittelrhein, Germany  ·  SCT + SCT Inst

Adds SCT Inst (SEPA Credit Transfer Instant, real-time, 10-second SLA) to the v1 SCT baseline. The request contract grows to accommodate real-time routing: serviceLevel and localInstrument are introduced. The failure response is unchanged from v1.

What changed from v1: Request adds serviceLevel and localInstrument. Response adds gcTransactionId and railName. errorCode remains optional with no guaranteed format or translation.
SCT Inst timeout gap: When the real-time rail does not respond within 10 seconds, the connector returns an empty response body. No timeout code is guaranteed. This was patched at Sparkasse with a bank-specific wrapper that lives in the Sparkasse build only and is not in this baseline.
Base URL https://api.grandcentral.backbase.io/v2
POST /payments/initiate Initiate a payment

Routes to SCT or SCT Inst based on paymentMethod. The core names the rail; the connector obeys. If SCT_INST is named and the rail does not respond within 10 seconds, the response body may be empty.

Known gaps - carried forward from v1, new in v2

From v1: errorCode has no guaranteed format. Core codes pass through raw. From v2: when SCT Inst times out (no response in 10 seconds), the connector returns an empty body. No timeout code is guaranteed under this contract. Patched at Sparkasse with a bank-specific wrapper not in this baseline. Both gaps are addressed in v3.5.
