Grand Central · iPaaS

Payments Connector

Version 1.0.0  ·  Apex Banking, Ireland  ·  SCT only

The first implementation of the Grand Central Payments Connector. Supports a single rail: SCT (SEPA Credit Transfer, standard, D+1 batch). The request contract mirrors ISO 20022 pain.001 field naming. The failure response returns status and an optional error code with no guaranteed format or translation.

Spec gap present from v1: errorCode is optional and may be empty, a raw scheme code, or a raw core code. No origin field. This contract defines the gap that v3.5 addresses.
Base URL https://api.grandcentral.backbase.io/v1
POST /payments/initiate Initiate a payment

Accepts a payment instruction from the core banking system and routes it to the SCT rail. The connector translates and submits. It does not make routing decisions.

Known gap - present from v1

The errorCode field is optional and has no guaranteed format. A raw T24 core code (e.g. EB-VAL-009) is as valid as a SEPA scheme code. An empty response is valid. No origin field tells the receiving system where in the flow the failure occurred. This gap was addressed in v1 at Apex Banking with a bank-specific response transformer that lives in the Apex build only.
